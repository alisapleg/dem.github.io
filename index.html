Сегодня я решила разгадать несколько шифров Цезаря — классических задач по криптографии. Вот мои решения:

1. Определение ключей шифра Цезаря

а. АПЕЛЬСИН → САЦЬНВЩЮ

Используем алфавит без «Ё» (32 буквы). Сравниваем первую букву: А (0) → С (17). Сдвиг = 17.
Проверка: П (15) + 17 = 32 ≡ 0 → А. Верно.
Ключ: k = 17

б. АБРИКОС → ЫЬЛГЕЙМ

А (0) → Ы (27). Сдвиг = 27.
Проверка: Б (1) + 27 = 28 → Ь. Верно.
Ключ: k = 27

2. Расшифровка сообщений

а. ФХНЗКЧ

Перебирая ключи, при k = 5 получаем:
Ф→П, Х→Р, Н→И, З→В, К→Е, Ч→Т → ПРИВЕТ

б. ЦЩЕБФ

При k = 20:
Ц→В, Щ→Е, Е→С, Б→Н, Ф→А → ВЕСНА

3. Шифрование фамилии

Фамилия: Палагина → ПАЛАГИНА
Ключ: k = 10

Применяем сдвиг:
П→Щ, А→К, Л→Х, А→К, Г→Н, И→С, Н→Ч, А→К

Результат: ЩКХКНСЧК

Все вычисления выполнены в 32-буквенном русском алфавите (без «Ё»):
А=0, Б=1, ..., Я=31.

Сегодня я решила разобраться, из каких частей состоит доменное имя моего сайта и как устроена его иерархия. Поскольку мой блог размещён на GitHub Pages, его адрес выглядит так: https://<имя-пользователя>.github.io.

Возьмём конкретный пример: если мой аккаунт на GitHub называется alyapalagina, то полное доменное имя сайта будет: alyapalagina.github.io.

Эта структура состоит из трёх уровней доменной иерархии (читается справа налево):

  • .io — домен верхнего уровня (TLD, Top-Level Domain). Изначально он принадлежал Британской территории в Индийском океане, но сейчас широко используется в IT-сфере.
  • github — домен второго уровня (SLD, Second-Level Domain). Он принадлежит компании GitHub и используется для предоставления поддоменов пользователям.
  • alyapalagina — поддомен третьего уровня, созданный автоматически при публикации репозитория через GitHub Pages.

Таким образом, DNS-система читает доменное имя от глобального к локальному: сначала определяется зона .io, внутри неё — серверы github.io, а уже внутри них — конкретный пользовательский сайт.

Это удобно, бесплатно и отлично подходит для личных блогов, документации или демонстрационных проектов!
